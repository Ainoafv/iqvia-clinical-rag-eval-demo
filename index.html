Core principle · 2 of 4

I treat AI outputs as hypotheses to validate — not results to trust.

The model proposes · the evaluation gate decides · the qualified reviewer approves

Production patterns from regulated healthcare AI: evaluation gates before release, claim-level grounding checks, structured JSON agent output, and append-only audit trails. Nothing ships on model confidence alone.

How to explore

  1. Play the scenario — 90-second guided walkthrough. Pipeline, panels, and charts animate automatically.
  2. Try question #2 — the eval trap. Watch groundedness drop to 50% and the gate block the answer.
  3. Inspect metrics — donut charts and claim bars update live with each query.
  4. Run golden tests — 5 regression tests that must pass before any model update ships.
How to use

Four panels to explore

  1. Play the scenario — top toolbar. Full walkthrough with narration, spotlight effects, and live charts.
  2. Panel 1 — Questions — pick a clinical query. #2 is the hallucination trap.
  3. Panel 2 — Pipeline — retrieval scores, claims, structured JSON, evaluation gate, approve/reject.
  4. Panel 3 — Metrics — donut charts, claim breakdown, golden tests, audit log.
